# 神经活动数据

此数据由清华大学自动化系[吴嘉敏老师](https://scholar.google.com/citations?user=4Z5PWEEAAAAJ&hl=zh-CN&oi=ao)团队提供，包含了小鼠大脑中 62130 个神经元在 28801 个时间步的活动数据。

原始数据为 Matlab 的 `.mat` 格式，已经过预处理转换为适合 Python 处理的 `.npy` 和 `.csv` 格式。
* `data.npy`：一个形状为 (28801, 62130) 的 NumPy 数组，表示每个神经元在每个时间步的活动强度。
* `node_info.csv`：一个 CSV 文件，包含每个神经元的具体信息，包括所在脑区和空间坐标。
* `time_info.csv`：一个 CSV 文件，包含每个时间步的具体信息，主要是刺激类型（清醒 Awake, 快速眼动睡眠 REM, 非快速眼动睡眠 NREM）。根据吴老师组内同学的说法，改标签由头皮脑电 (EEG) 给出，可认为是真值。此外，根据吴老师组内同学的说法，采样频率约为 4Hz。

## 熟悉数据 - 尝试建模

本数据可以被看作一个大规模网络动力学系统。我们的最终建模目标，是尝试找到每个神经元活动随时间演化的动力学方程：

$$
\frac{\mathrm{d}}{\mathrm{d}t}x_i(t) = f_i(\mathbf{x}(t)),
$$

其中，$`x_i(t)`$ 表示第 $`i`$ 个神经元在时刻 $`t`$ 的激活强度，$`\mathbf{x}(t)`$ 表示所有神经元在该时刻的整体活动状态。

由于当前数据是离散采样的，上式在实际建模中更接近于：

$$
x_i(t+1) = x_i(t) + f_i(\mathbf{x}(t)),
$$

也就是说，模型需要根据当前所有神经元的状态 $`\mathbf{x}(t)`$，预测下一时刻第 $`i`$ 个神经元的变化量：

$$
x_i(t+1) - x_i(t).
$$

一个合理的动力学模型应当至少具有一定的预测能力。预测能力可以分为两类：

1. **单步预测能力**：模型输出的 $`f_i(\mathbf{x}(t))`$ 应当与真实变化量 $`x_i(t+1)-x_i(t)`$ 有较强相关性。
2. **长期预测能力**：将模型递推多步后得到的 $`x_i(t+n)`$，应当与对应的真实观测值保持较好的相关性。

根据吴老师组内同学所述，目前较好的 AI 模型大约可以在数秒尺度的长期预测中保持相关性；再长时间后，预测结果通常会逐渐偏离真实轨迹。另一方面，吴老师本人也提到，只要模型具有较好的单步预测能力，长期精确预测并不是最重要的目标。相比之下，研究者更关心模型是否能够恢复宏观层面的群体活动模式，例如上面的动图中展示出的活动模式。不过，这类宏观行为目前不太容易直接量化，因此暂时不作为主要评估指标。还有，据吴老师组内同学表示，直接使用全部 62130 个神经元进行建模效率很低，而且可能并不必要，因为大脑中有大量神经元具有相似的活动模式和功能。因此，在建模时可以先从部分神经元入手（例如随机采样数百个神经元）以提高效率和可解释性。

此外在建模时还需要注意，当前数据中并不包含神经元之间的连接关系或连接权重。因此，如果希望将神经元建模为相互影响的动力学系统，需要通过某种方式预先估计神经元之间的连接结构，相关思路见后文讨论。


### 神经动力学

作为熟悉数据的一环，可以先尝试用不同神经网络和不同建模方案拟合神经动力学。这里暂时不要求模型具有很强的可解释性，而是将动力学项看作一个黑盒函数：

$$
\mathbf{x}(t+1) = \mathbf{x}(t) + f_{\Phi}(\mathbf{x}(t)),
$$

其中，$`\mathbf{x}(t)`$ 表示所有神经元在时刻 $`t`$ 的活动状态，$`f_{\Phi}`$ 是由参数 $`\Phi`$ 表示的神经网络模型。模型的目标是根据当前或历史神经活动，预测下一时刻的神经活动变化。

可以尝试的模型结构包括但不限于：

| 模型 | 建模思路 |
|---|---|
| MLP | 将当前时刻的神经活动 $`\mathbf{x}(t)`$ 作为输入，直接预测下一步变化量。 |
| RNN (GRU / LSTM) | 输入多个历史时间步，利用循环结构建模时间依赖关系。 |
| GNN (GCN / Message Passing GNN) | 将神经元看作图上的节点，通过节点之间的信息传递建模神经元之间的相互影响。 |
| Transformer | 将神经元状态视为 token，利用注意力机制建模神经元之间的关系。 |

在时间建模上，可以尝试不同的历史步数 `hist_steps`。例如，只使用当前时刻，或使用历史 2 步、历史 3 步来预测下一步；对于 RNN 这类模型，也可以尝试使用更长的历史序列：

$$
\{\mathbf{x}(t-\mathrm{hist\_steps}+1), \ldots, \mathbf{x}(t)\}
\rightarrow \mathbf{x}(t+1).
$$

在神经元之间关系的建模上，可以从简单到复杂逐步尝试。最简单的方案是不显式考虑神经元之间的相关性，即每个神经元只根据自身历史预测自身未来：

$$
x_i(t+1) = x_i(t) + f_{\Phi}(x_i(t)).
$$

进一步地，也可以考虑所有神经元之间的相互影响，例如使用 MLP 或 Transformer 直接建模整体状态：

$$
\mathbf{x}(t+1) = \mathbf{x}(t) + f_{\mathrm{MLP}}(\mathbf{x}(t)),
$$

或

$$
\mathbf{x}(t+1) = \mathbf{x}(t) + f_{\mathrm{Transformer}}(\mathbf{x}(t)).
$$

如果使用 GNN，可以将神经元看作图上的节点，令边表示神经元之间可能存在的相互作用。一种可能的形式是：

$$
x_i(t+1) = x_i(t) + \sum_j A_{ij} f_{\mathrm{GNN}}(x_i(t), x_j(t)),
$$

其中，$`A_{ij}`$ 表示节点 $`j`$ 对节点 $`i`$ 的影响强度或连接权重。需要注意的是，当前数据中并没有直接提供神经元之间的连接关系，因此网络拓扑 $`A_{ij}`$ 需要自行设计或估计。可以尝试的拓扑构建方式包括但不限于：

| 拓扑构建方式 | 说明 |
|---|---|
| 基于时序相关性 | 如果两个神经元的活动轨迹相似，则认为它们之间可能存在更强连边。 |
| 基于空间距离 | 空间位置更接近的神经元之间设置更强连接或更高连边概率。 |
| 可学习节点嵌入 | 为每个神经元分配一个可学习嵌入，用嵌入之间的相似性作为连接强度。 |
| 混合拓扑 | 同时结合空间距离、脑区标签、时序相关性和可学习参数构建图结构。 |

这部分实验的主要目的，是快速了解数据中可预测的时间结构和神经元之间可能存在的相互依赖关系。后续如果希望获得更强的可解释性，可以进一步考虑白盒动力学模型。

### 白盒动力学

使用 [ND2](https://github.com/yuzhTHU/ND2) 方法搜索数学方程。例如，一种可能的方程形式是：

$$
x_i(t+1) - x_i(t) = F(x_i(t)) + \sum_j A_{ij} G(x_i(t), x_j(t)),
$$

其中，$`F`$ 表示神经元自身的动力学，$`G`$ 表示神经元之间的相互作用，$`A_{ij}`$ 表示连接权重。通过 ND2 方法搜索 $`F`$、$`G`$ 的具体形式，可以得到一个具有一定可解释性的白盒动力学模型。需要注意的是，ND2 的搜索效率不支持直接使用全部 62130 个节点进行搜索，因此可能需要先对神经元进行采样，或者先使用黑盒模型筛选出具有较强预测能力的神经元子集，再在该子集上使用 ND2 进行建模。除了 ND2 以外，也可以尝试其他符号回归方法，例如 [适用于网络动力学的 SINDy 方法](https://codeocean.com/capsule/0776097/tree/v1) （访问链接时记得关掉梯子）等等。

在时间建模上，同样可以尝试不同的历史步数 `hist_steps`，例如只使用当前时刻，或使用历史 2 步、历史 3 步来预测下一步。这大致等价于在方程中引入不同阶数的时间导数项，例如：

$$
x_i(t+1) - x_i(t) = F(x_i(t), x_i(t-1), \ldots) + \sum_j A_{ij} G(x_i(t), x_i(t-1), \ldots, x_j(t), x_i(t-1), x_j(t-1), \ldots).
$$

在神经元之间关系的建模上，同样可以从简单到复杂逐步尝试。ND2 需要以相互作用矩阵 $`A_{ij}`$ 作为输入，因此可以先使用一个简单的拓扑结构，例如空矩阵或单位矩阵，来搜索不考虑神经元之间相关性的动力学方程：

$$
x_i(t+1) - x_i(t) = F(x_i(t), x_i(t-1), \ldots).
$$

进一步地，也可以考虑所有神经元之间的相互影响，例如使用一个全一矩阵 $`A_{ij}`$ 矩阵来搜索整体状态的动力学方程：

$$
x_i(t+1) - x_i(t) = F(x_i(t), x_i(t-1), \ldots) + \sum_j A_{ij} G(x_i(t), x_i(t-1), \ldots, x_j(t), x_i(t-1), x_j(t-1), \ldots).
$$

除此之外，还可以尝试基于时序相关性、空间距离、可学习节点嵌入等方式构建不同的 $`A_{ij}`$ 矩阵，来搜索不同结构的动力学方程。

值得注意的是，使用不同的矩阵会得到不同的方程结构和拟合效果。因此，当某个建模方案得到的方程效果不好时，可以尝试更换不同的矩阵来搜索，看看是否能够得到更好的方程结构和拟合效果。通过这种方式，可以在一定程度上探索神经元之间的相互关系以及它们对整体动力学的影响。


### 模型评价

为了方便不同算法之间进行比较，`scripts/neural/evaluate.py` 中已经提供了一个统一的评估脚本。该脚本会自动完成数据加载、采样、训练、预测和指标统计等流程。上面实现的神经动力学模型和白盒动力学模型都应该包装后使用该评估脚本进行测试，以便相互进行横向比较。

该脚本的评估流程大致如下：

1. **加载数据**：读取神经活动数据、神经元信息和时间标签。目前加载约 6G 的 `data.csv` 大约需要 30s，后续可能进一步优化。
2. **采样节点**：使用固定随机种子从全部神经元中选择 `n_nodes` 个节点，避免直接使用全部 62130 个神经元导致计算过慢。目前默认随机采样 256 个节点，其中一半用于训练，一般用于测试。后续可能增加更合理的采样方式，例如根据脑区标签分层采样，或根据神经元之间的相关性进行聚类后采样。
3. **选择状态类型**：可以指定使用特定的 `stimuli` 类型，例如 `all`、`rem`、`nrem` 或 `awake`，只提取对应状态下的时间步进行训练和测试。
4. **划分训练/测试区域**：将数据沿时间维度和节点维度划分为四个象限：
   - 第一象限：训练模型；
   - 第二象限：测试同一批节点、不同时间范围上的泛化能力；
   - 第三象限：测试不同批节点、同一时间范围上的泛化能力（对于 GNN 等依赖于神经元之间连接关系 $`A_{ij}`$ 的模型，该评估脚本直接将第一象限中估计得到的 $`A_{ij}`$ 用于测试。这几乎必然导致效果较差。后续可能会对此评估脚本进行改进，允许算法先基于这段数据估计网络结构，再测试其性能）
   - 第四象限：测试不同批节点、不同时间范围上的泛化能力（同上。第三、第四象限的性质比较接近，后续可能考虑合并。）
5. **训练模型**：将训练数据传入算法模块，训练得到模型。具体接口可以参考 `scripts/neural/algorithms/linear.py` 中的 `train_model`。
6. **构造预测函数**：算法需要返回一个预测函数，根据给定的 `hist_steps` 个历史时间步预测未来一步，即根据
   \[
   \{\mathbf{x}(t-\mathrm{hist\_steps}+1), \ldots, \mathbf{x}(t)\}
   \]
   预测 $`\mathbf{x}(t+1)`$。具体接口可以参考 `scripts/neural/algorithms/linear.py` 中的 `get_predict_func`。
7. **测试预测效果**：将三个测试象限的数据分别传入预测函数，比较预测值与真实值之间的相关性。
8. **统计长期预测能力**：通过递推预测，计算模型在多长时间尺度内仍能保持较好的相关性，例如相关性下降到特定阈值之前可以维持多少秒。
9. **保存结果**：打印并保存评估日志，便于不同算法之间进行横向比较。

为了测试自己的算法，请参考 `scripts/neural/algorithms/linear.py` 新建一个算法文件，例如：

```bash
scripts/neural/algorithms/my_algorithm.py
```

然后运行：

```bash
python ./scripts/neural/evaluate.py --algorithm my_algorithm
```

算法文件需要实现与 `linear.py` 中相同的接口，使评估脚本能够调用其训练函数和预测函数。

附：当前评估脚本中主要使用以下指标

| 指标            | 含义                                                  |
| ------------- | --------------------------------------------------- |
| `pearson`     | 将所有预测值和真实值展平后计算 Pearson 相关系数，用于衡量预测结果与真实活动模式的线性相关性。 |
| `r2`          | 将所有预测值和真实值展平后计算 $`R^2`$ 分数，用于衡量预测值对真实变化的解释能力。         |
| `step`        | 递推预测的步数。`step = 0` 表示初始真实输入，`step = 1` 表示单步预测，`step = n` 表示递推 n 步。      |
| `seconds`     | 当前递推步数对应的实际时间，计算方式为 `step / sampling_hz`。           |
| `n_time`      | 当前评估中可用的时间窗口数量。随着递推步数增加，可用连续窗口可能减少。                 |
| `n_node`      | 当前评估中使用的神经元数量。                                      |
| `pearson>0.5` | Pearson 相关性能够维持在 0.5 以上的最长预测步数和对应时间。                |
| `r2>0.5`      | $`R^2`$ 能够维持在 0.5 以上的最长预测步数和对应时间。                     |
