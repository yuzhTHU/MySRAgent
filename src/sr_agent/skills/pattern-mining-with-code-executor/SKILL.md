# Pattern Mining with code_executor

## Purpose

This skill helps the agent use `code_executor` to understand the data before proposing more formulas.

The goal is to discover useful structure in the dataset, so that later calls to `evaluate_formula` are more targeted and less random.

## When to Use

Use this skill when:

- The relationship between features and target is unclear.
- Several evaluated formulas have poor MSE.
- The current best formula seems accidental or overly complex.
- You suspect hidden structure such as products, ratios, powers, logs, exponentials, or periodicity.
- You need to inspect data statistics, correlations, residuals, or transformations.
- You are stuck and need a new hypothesis.

## Guidance

Use `code_executor` to look for patterns such as:

- whether the target is linear in one feature
- whether the target is polynomial in one or more features
- whether the target depends on a product of features
- whether the target depends on a ratio of features
- whether log, exp, sqrt, square, inverse, sin, or cos transformations help
- whether the target is separable into additive or multiplicative parts
- whether residuals show a pattern after subtracting a candidate formula
- whether some features are irrelevant
- whether the data has constant, monotonic, symmetric, or periodic behavior

## Suggested Analyses

Useful code-based checks include:

1. Print basic statistics for each feature and target.
2. Compute correlations between features and target.
3. Try simple transformed features:
   - `x`
   - `x^2`
   - `sqrt(x)`
   - `1/x`
   - `log(x)` when valid
   - `exp(x)` when numerically safe
   - `sin(x)`, `cos(x)` when periodicity is plausible
4. Check feature interactions:
   - `x1 + x2`
   - `x1 - x2`
   - `x1 * x2`
   - `x1 / x2`
   - powers of one feature times another feature
5. Fit simple linear regressions on transformed features to identify candidate structures.
6. Inspect residuals from the current best formula.
7. Look for exact or near-exact relationships.

## How to Use Results

After using `code_executor`, summarize the discovered pattern before calling `evaluate_formula`.

Good summaries look like:

- “The target appears linear in `x1^2` and `x2`.”
- “The residual after subtracting `x1 * x2` is approximately sinusoidal in `x3`.”
- “The target seems to depend on the ratio `x1 / x2` rather than on `x1` and `x2` separately.”
- “Feature `x4` appears irrelevant.”
- “A log transform of the target makes the relationship nearly linear.”

Then propose formulas that directly test those observations.

## Common Mistakes

Avoid these behaviors:

- Using `code_executor` only to print raw data without analysis.
- Trying many transformations without summarizing what was learned.
- Jumping from weak correlation directly to a complex formula.
- Ignoring numerical validity, such as taking `log(x)` when `x <= 0`.
- Overfitting transformed features without checking whether the resulting formula is simple.
- Calling `evaluate_formula` immediately after code execution without using the discovered pattern.

## Recommended Pattern

A good loop is:

1. Use `code_executor` to inspect candidate structures.
2. Identify one or two promising transformations.
3. State the hypothesis clearly.
4. Convert the hypothesis into a simple formula.
5. Call `evaluate_formula`.
6. If the result improves, refine or simplify.
7. If the result fails, return to pattern mining with a different hypothesis.
