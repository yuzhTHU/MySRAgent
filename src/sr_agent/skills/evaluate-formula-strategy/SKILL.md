# Evaluate Formula Strategy

## Purpose

This skill helps the agent use `evaluate_formula` more strategically during symbolic regression.

The goal is not to evaluate many formulas randomly, but to evaluate a small number of informative candidate formulas that can improve the search direction.

## When to Use

Use this skill when:

- You are about to call `evaluate_formula`.
- Several formulas have already been evaluated.
- The best formula has stopped improving.
- Multiple candidate formulas look similar.
- A formula has good accuracy but is too complex.
- You need to decide whether to exploit a promising structure or explore a new one.

## Guidance

Before calling `evaluate_formula`, ask:

1. Is this formula structurally different from previous formulas?
2. Is this formula simpler than the current best formula?
3. Does this formula test a clear hypothesis about the data?
4. Does this formula reuse useful information from previous failed attempts?
5. Is there a cheaper analysis step that should be done first?

Prefer evaluating formulas that test specific hypotheses, such as:

- linearity
- polynomial structure
- ratios between variables
- products or interactions between variables
- logarithmic or exponential relationships
- trigonometric or periodic structure
- separability across variables
- known physical or dimensional structure

## Evaluation Strategy

When proposing candidate formulas:

- Start with simple formulas.
- Change one structural idea at a time.
- Compare against the current best formula.
- Avoid evaluating algebraically equivalent formulas.
- Prefer interpretable improvements over small accuracy gains from complexity.
- If the formula is complex, try a simpler approximation next.

When interpreting results:

- MSE is the main optimization target.
- RMSE is useful for understanding error scale.
- MAE is useful for robustness against outliers.
- R² is useful, but should not override simplicity.
- A lower MSE is not always better if the formula becomes unnecessarily complex.

## Common Mistakes

Avoid these behaviors:

- Calling `evaluate_formula` on many random formulas without a hypothesis.
- Repeating formulas that are only minor algebraic rearrangements.
- Making formulas more complex after every failure.
- Ignoring previously evaluated formulas in the context.
- Treating a high R² as sufficient when MSE is still not near zero.
- Accepting a complex formula when a simpler exact or near-exact structure may exist.

## Recommended Pattern

A good loop is:

1. Inspect the best formulas and their errors.
2. Identify what structural assumption has not been tested.
3. Propose one or a few formulas that test that assumption.
4. Call `evaluate_formula`.
5. Use the result to decide whether to refine, simplify, or switch strategy.
